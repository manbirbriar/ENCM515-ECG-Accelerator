Project Overview (40 Marks)
This Group Research Project challenges you to design, implement, analyze, and critically
evaluate an end-to-end application-specific processor or accelerator-oriented computing framework aligned with the learning outcomes of ENCM 515: ApplicationSpecific Processors and Accelerators.
Your project must integrate the core concepts of the course (where appropriate): processor architecture and execution models, instruction-level and data-level parallelism, DSPoriented workloads, precision-aware computation, and hardware–software co-design principles. Projects should focus on real-world domains such as wearable biomedical systems,
autonomous drones, smart cameras, augmented reality devices, IoT inference engines, financial data streams, seismic monitoring, or multimedia processing pipelines.
Projects must combine architectural reasoning, performance modeling, structured optimization analysis, and accelerator-oriented system-level evaluation. In
addition, your work MUST include a working coding-based simulation that demonstrates and validates your accelerator-oriented ideas using MATLAB and/or Python
and/or Cython and/or MicroPython (a single tool or a justified combination is acceptable). Your simulation must be used to generate measurable performance or behavioral
results (not only diagrams or conceptual discussion).
Based on your chosen topic, your work MUST integrate key elements such as:
• Architectural analysis of execution models (e.g., RISC vs DSP vs application-specific
accelerators, pipeline organization, SIMD/vector processing, superscalar or VLIW concepts);
• Simulation-based modeling of datapath behavior, instruction flow, and pipeline characteristics (e.g., latency, initiation interval, throughput, hazard analysis) using MATLAB/Python/Cython/MicroPython where appropriate;
• Performance evaluation using quantitative metrics (e.g., CPI, speedup, Amdahl’s Law,
memory bandwidth impact, resource constraints) supported by simulation results;

Learning Outcomes
By completing this project, you will be able to:
• Analyze and compare processor and accelerator architectures (e.g., RISC, DSP-oriented,
SIMD, superscalar, VLIW) and justify design choices based on application constraints.
• Model instruction execution, datapath behavior, and pipeline characteristics (e.g., hazards, initiation interval, latency, throughput) using simulation-based approaches.
• Implement and evaluate accelerator-oriented workloads using MATLAB, Python,
Cython, and/or MicroPython, and generate quantitative performance results.
• Apply hardware- and software-level optimization techniques (e.g., loop unrolling, fusion/fission, pipelining, scheduling, data reuse) and measure their performance impact.
• Evaluate memory hierarchy effects, bandwidth limitations, and data locality strategies
in computational workloads.
